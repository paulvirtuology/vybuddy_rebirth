# Procédure de Support pour les Systèmes de Réservation de Salles

**Catégorie:** meeting_rooms
**Description:** Procédure pour diagnostiquer et résoudre les problèmes liés aux systèmes de réservation de salles de réunion.

## Questions de diagnostic

- Quel est le problème spécifique rencontré avec le système de réservation?
- L'utilisateur a-t-il déjà utilisé le système de réservation auparavant?
- Le problème concerne-t-il une salle spécifique ou toutes les salles?
- L'utilisateur a-t-il reçu un message d'erreur? Si oui, lequel?
- L'utilisateur a-t-il essayé de redémarrer l'application ou le système?

## Étapes de résolution

### Étape 1: Vérifier l'accès utilisateur

Assurez-vous que l'utilisateur a les droits d'accès nécessaires pour réserver des salles.

### Étape 2: Vérifier la configuration du système de réservation (nécessite droits admin)

Confirmez que le système de réservation est correctement configuré pour l'utilisateur et la salle concernée.

### Étape 3: Tester la réservation

Effectuez un test de réservation pour vérifier si le problème persiste.

### Étape 4: Redémarrer le système

Demandez à l'utilisateur de redémarrer l'application de réservation ou le système.

### Étape 5: Escalader au support N2

Si le problème persiste après les étapes précédentes, escalader au support N2.


## Création de ticket Odoo

**Quand créer un ticket:** Créer un ticket Odoo si le problème n'est pas résolu après les étapes de résolution ou si une intervention administrative est nécessaire.

**Champs requis:**
- **title:** Problème de réservation de salle - [Nom de l'utilisateur] - [Date]
- **description:** Inclure une description détaillée du problème, les étapes de diagnostic effectuées, et toute capture d'écran pertinente.
- **priority:** Moyenne

## Problèmes fréquents

- Problème d'accès au système de réservation
- Erreur lors de la tentative de réservation
- Salle non disponible malgré la disponibilité affichée
- Problème de synchronisation avec le calendrier
